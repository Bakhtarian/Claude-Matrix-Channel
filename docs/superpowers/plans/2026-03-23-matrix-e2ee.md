# Matrix E2EE Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add transparent E2EE to the Matrix channel so it works in any room — encrypted or unencrypted — with full interactive SAS verification.

**Architecture:** Replace `@matrix-org/matrix-sdk-crypto-nodejs` with `@matrix-org/matrix-sdk-crypto-wasm`, managing `OlmMachine` directly. Subclass `MatrixClient` to intercept sync responses and feed crypto data to `OlmMachine` before the bot-sdk processes room events. Encrypt/decrypt transparently; enforce verified-device-only communication in encrypted rooms.

**Tech Stack:** TypeScript, Bun, `matrix-bot-sdk@0.7.1`, `@matrix-org/matrix-sdk-crypto-wasm@^18.0.0`, `@modelcontextprotocol/sdk`

**Spec:** `docs/superpowers/specs/2026-03-23-matrix-e2ee-design.md`

```typescript
/**
 * E2EE crypto layer using @matrix-org/matrix-sdk-crypto-wasm.
 *
 * Manages OlmMachine lifecycle, sync integration, and outgoing crypto
 * request dispatch. Used by CryptoMatrixClient (subclass of MatrixClient)
 * to intercept sync responses and feed them to the OlmMachine.
 */

import {
  OlmMachine, UserId, DeviceId, RoomId,
  DeviceLists, EncryptionSettings, CollectStrategy,
  DecryptionSettings, TrustRequirement, RequestType,
  VerificationMethod, ShieldColor,
  initAsync,
  type OutgoingRequest,
} from '@matrix-org/matrix-sdk-crypto-wasm'
import { MatrixClient, SimpleFsStorageProvider } from 'matrix-bot-sdk'
import { join } from 'path'

let machine: OlmMachine | null = null

export function getOlmMachine(): OlmMachine {
  if (!machine) throw new Error('OlmMachine not initialized')
  return machine
}

export async function initCrypto(
  botUserId: string,
  deviceId: string,
  storePath: string,
): Promise<OlmMachine> {
  await initAsync()
  machine = await OlmMachine.initialize(
    new UserId(botUserId),
    new DeviceId(deviceId),
    storePath,
  )
  process.stderr.write(`matrix channel: E2EE initialized (device: ${deviceId})\n`)
  return machine
}

export function closeCrypto(): void {
  if (machine) {
    machine.close()
    machine = null
  }
}

// --- Outgoing crypto request dispatch ---

const REQUEST_ENDPOINTS: Record<number, { method: string; path: (req: any) => string }> = {
  [RequestType.KeysUpload]: {
    method: 'POST',
    path: () => '/_matrix/client/v3/keys/upload',
  },
  [RequestType.KeysQuery]: {
    method: 'POST',
    path: () => '/_matrix/client/v3/keys/query',
  },
  [RequestType.KeysClaim]: {
    method: 'POST',
    path: () => '/_matrix/client/v3/keys/claim',
  },
  [RequestType.ToDevice]: {
    method: 'PUT',
    path: (req: any) =>
      `/_matrix/client/v3/sendToDevice/${encodeURIComponent(req.event_type)}/${encodeURIComponent(req.txn_id)}`,
  },
  [RequestType.SignatureUpload]: {
    method: 'POST',
    path: () => '/_matrix/client/v3/keys/signatures/upload',
  },
  [RequestType.RoomMessage]: {
    method: 'PUT',
    path: (req: any) =>
      `/_matrix/client/v3/rooms/${encodeURIComponent(req.room_id)}/send/${encodeURIComponent(req.event_type)}/${encodeURIComponent(req.txn_id)}`,
  },
}

export async function sendCryptoRequest(
  client: MatrixClient,
  req: OutgoingRequest,
): Promise<string> {
  const endpoint = REQUEST_ENDPOINTS[req.type]
  if (!endpoint) {
    process.stderr.write(`matrix channel: unknown crypto request type: ${req.type}\n`)
    return '{}'
  }
  const body = JSON.parse(req.body)
  const response = await client.doRequest(
    endpoint.method,
    endpoint.path(req),
    null, // qs
    body,
  )
  return JSON.stringify(response)
}

export async function processOutgoingRequests(client: MatrixClient): Promise<void> {
  const m = getOlmMachine()
  const requests = await m.outgoingRequests()
  for (const req of requests) {
    try {
      const response = await sendCryptoRequest(client, req)
      await m.markRequestAsSent(req.id, req.type, response)
    } catch (err) {
      process.stderr.write(`matrix channel: crypto request failed (type ${req.type}): ${err}\n`)
    }
  }
}

// --- Sync data feed ---

export async function feedCryptoSync(raw: any): Promise<void> {
  const m = getOlmMachine()
  const toDeviceEvents = raw?.to_device?.events || []
  const changed = (raw?.device_lists?.changed || []).map((u: string) => new UserId(u))
  const left = (raw?.device_lists?.left || []).map((u: string) => new UserId(u))
  const deviceLists = new DeviceLists(changed, left)

  const otkeyCounts = new Map<string, number>()
  for (const [algo, count] of Object.entries(raw?.device_one_time_keys_count || {})) {
    otkeyCounts.set(algo, count as number)
  }

  const unusedFallbacks = raw?.device_unused_fallback_key_types
    ? new Set<string>(raw.device_unused_fallback_key_types)
    : undefined

  const processed = await m.receiveSyncChanges(
    JSON.stringify(toDeviceEvents),
    deviceLists,
    otkeyCounts,
    unusedFallbacks,
    new DecryptionSettings(TrustRequirement.Untrusted),
  )

  // Check for incoming verification requests in processed to-device events
  for (const evt of processed) {
    try {
      const raw = JSON.parse(evt.rawEvent)
      if (raw?.type === 'm.key.verification.request') {
        const sender = raw.sender as string
        process.stderr.write(
          `matrix channel: incoming verification request from ${sender} — run /matrix:verify to accept\n`,
        )
      }
    } catch {}
  }
}

// --- CryptoMatrixClient subclass ---

export class CryptoMatrixClient extends MatrixClient {
  protected async processSync(
    raw: any,
    emitFn?: (emitEventType: string, ...payload: any[]) => Promise<any>,
  ): Promise<any> {
    // Feed crypto data BEFORE bot-sdk processes room events.
    // This ensures OlmMachine has session keys when we decrypt timeline events.
    try {
      await feedCryptoSync(raw)
      await processOutgoingRequests(this)
    } catch (err) {
      process.stderr.write(`matrix channel: crypto sync failed: ${err}\n`)
    }

    return super.processSync(raw, emitFn)
  }
}

// --- Room encryption helpers ---

export async function isRoomEncrypted(
  client: MatrixClient,
  roomId: string,
): Promise<boolean> {
  try {
    const state = await client.getRoomStateEvent(roomId, 'm.room.encryption', '')
    return !!state?.algorithm
  } catch {
    return false
  }
}
```

```typescript
export type DecryptionResult = {
  event: Record<string, unknown>
  shieldColor: number // ShieldColor enum value
  shieldMessage?: string
}

export async function decryptRoomEvent(
  roomId: string,
  event: Record<string, unknown>,
): Promise<DecryptionResult> {
  const m = getOlmMachine()
  const decrypted = await m.decryptRoomEvent(
    JSON.stringify(event),
    new RoomId(roomId),
    new DecryptionSettings(TrustRequirement.Untrusted),
  )
  const shield = decrypted.shieldState(true) // strict mode
  const clearEvent = JSON.parse(decrypted.event)
  return {
    event: clearEvent,
    shieldColor: shield.color,
    shieldMessage: shield.message ?? undefined,
  }
}
```

```typescript
// --- Encryption ---

export async function encryptIfNeeded(
  client: MatrixClient,
  roomId: string,
  eventType: string,
  content: Record<string, unknown>,
): Promise<{ eventType: string; content: Record<string, unknown> }> {
  const encrypted = await isRoomEncrypted(client, roomId)
  if (!encrypted || !machine) {
    return { eventType, content }
  }

  const m = getOlmMachine()

  // Ensure we're tracking room members
  const members = await client.getJoinedRoomMembers(roomId)
  const userIds = members.map((uid: string) => new UserId(uid))
  await m.updateTrackedUsers(userIds)

  // Claim missing one-time keys
  const missingSessions = await m.getMissingSessions(userIds)
  if (missingSessions) {
    const resp = await client.doRequest('POST', '/_matrix/client/v3/keys/claim', null, JSON.parse(missingSessions.body))
    await m.markRequestAsSent(missingSessions.id, missingSessions.type, JSON.stringify(resp))
  }

  // Share room key
  const settings = new EncryptionSettings()
  settings.sharingStrategy = CollectStrategy.onlyTrustedDevices()
  const shareRequests = await m.shareRoomKey(new RoomId(roomId), userIds, settings)
  for (const req of shareRequests) {
    await sendCryptoRequest(client, req)
  }

  // Encrypt the event
  const encryptedPayload = await m.encryptRoomEvent(
    new RoomId(roomId),
    eventType,
    JSON.stringify(content),
  )
  return {
    eventType: 'm.room.encrypted',
    content: JSON.parse(encryptedPayload),
  }
}
```

```typescript
async function sendEventEncrypted(
  roomId: string,
  eventType: string,
  content: Record<string, unknown>,
): Promise<string> {
  try {
    const encrypted = await encryptIfNeeded(matrixClient, roomId, eventType, content)
    return await matrixClient.sendEvent(roomId, encrypted.eventType, encrypted.content)
  } catch (err) {
    // If encryption fails (e.g., unverified devices), fall back to plaintext notice
    const msg = err instanceof Error ? err.message : String(err)
    if (msg.includes('trust') || msg.includes('verified') || msg.includes('encrypt')) {
      process.stderr.write(`matrix channel: encryption failed, sending plaintext notice: ${msg}\n`)
      return await matrixClient.sendEvent(roomId, 'm.room.message', {
        msgtype: 'm.notice',
        body: `[Encryption failed: ${msg}. Verify devices with /matrix:verify]`,
      })
    }
    throw err
  }
}
```

```markdown
# Matrix Device Verification

Interactive SAS (Short Authentication String) verification for E2EE.

## What This Does

Establishes cryptographic trust between the bot's device and another user's device so that encrypted messages can be exchanged. Both sides compare a set of 7 emoji — if they match, the devices are mutually verified.

## Steps

1. **Check for pending verification requests** — call the `getVerificationRequests` method on the OlmMachine for all known users. List any pending requests with their user ID, device ID, and time remaining.

2. **If no pending requests and no argument given** — show the bot's device ID and Ed25519 key, and instruct the user to initiate verification from Element (Settings > Sessions > the bot's device > Verify).

3. **If a user ID argument is given** — initiate verification toward that user:
   - Call `machine.getDevice(userId, deviceId)` to find their device
   - Call `device.requestVerification([VerificationMethod.SasV1])` to start
   - Send the resulting ToDeviceRequest via the crypto request dispatch

4. **Accept and start SAS** — once a request is ready:
   - Call `request.acceptWithMethods([VerificationMethod.SasV1])`
   - Send the resulting outgoing request
   - Call `request.startSas()` to begin the SAS exchange
   - Send the SAS outgoing request
   - If we are the acceptor, also call `sas.accept()` and send that request

5. **Display emoji** — poll `sas.canBePresented()`, then call `sas.emoji()` to get 7 emoji objects with `symbol` and `description` fields. Display them clearly:
   ```
   Verify these emoji match what you see in Element:

   🐶 Dog  |  🔑 Key  |  🎩 Hat  |  📌 Pin  |  🎧 Headphones  |  ✂️ Scissors  |  🔔 Bell

   Do they match? (yes/no)
   ```

6. **Confirm or cancel** — if the user confirms, call `sas.confirm()` and send the resulting requests. If they deny, call `sas.cancel()`.

7. **Process any remaining outgoing requests** after confirmation to complete the verification.

## Important

- Verification requests have a timeout (typically 10 minutes). Show `request.timeRemainingMillis()`.
- After verification completes, the OlmMachine automatically marks the device as trusted. No manual trust setting needed.
- The skill should import and use the crypto module functions from `crypto.ts`.
```
